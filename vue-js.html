Whizmath: Mastering Vue.js

Building Progressive, Performant & Maintainable User Interfaces

‚ú® Welcome to the World of Vue.js!

Hello, future Vue developers! Welcome to this comprehensive guide on Vue.js, powered by Whizmath. Vue.js is a progressive JavaScript framework used for building user interfaces. What does "progressive" mean? It means you can adopt Vue incrementally. You can use it just for a small interactive part of your page, or scale it up to build complex single-page applications.

Vue is designed to be approachable and easy to learn for beginners, while also powerful enough for advanced applications. It focuses on reactivity, components, and a clear separation of concerns (HTML templates, JavaScript logic, CSS styling). This lesson will take you from the fundamental concepts to some advanced patterns, providing clear explanations and practical examples. Let's start building amazing UIs with Vue!

üí° What Exactly is Vue.js?

Vue.js is a JavaScript framework for building user interfaces. It builds on standard HTML, CSS, and JavaScript and provides a declarative and component-based programming model that helps you efficiently develop user interfaces, whether they are simple or complex.

  • **Progressive Framework:** Can be integrated into an existing project incrementally or used for a full-scale single-page application.
  • **Approachable:** Easy to pick up for developers coming from HTML, CSS, or basic JavaScript.
  • **Performant:** Offers a lightweight Virtual DOM and optimized rendering.
  • **Versatile:** Suitable for small components, medium-sized applications, and large-scale enterprise solutions.
  • **Reactive:** Vue's reactivity system automatically tracks changes to your data and efficiently updates the DOM when data changes.

Vue provides a clear and organized way to manage your application's data and how it interacts with the UI, making it a joy to work with.

The Vue Application Instance

Every Vue application starts by creating a new **application instance** using `createApp()`. This instance acts as the root of your Vue application and mounts to a specific DOM element.

Basic Vue App Instance 
<!-- In your HTML, an element for Vue to mount to -->
<div id="my-app">
    <p>{{ message }}</p>
</div>

<script>
    const { createApp } = Vue; // Destructure createApp from the global Vue object

    createApp({
        // data option to define reactive data
        data() {
            return {
                message: 'Hello from Vue.js!'
            };
        }
    }).mount('#my-app'); // Mount the app to the #my-app element
</script>

The `{{ message }}` syntax is Vue's text interpolation, which allows you to display reactive data from your Vue instance directly in the template.

Directives (`v-`)

Vue directives are special attributes with the `v-` prefix. They apply special reactive behavior to the rendered DOM.

`v-bind`: Binding Attributes

Dynamically bind one or more attributes, or a component prop to an expression. Shorthand: `:attribute`.

v-bind Example 
<!-- In HTML -->
<div id="app-vbind">
    <img :src="imageUrl" :alt="imageAlt" class="rounded-lg shadow-md w-32 h-32 object-cover">
    <a :href="linkUrl" class="text-blue-400 hover:underline">Visit Whizmath!</a>
</div>

<script>
    createApp({
        data() {
            return {
                imageUrl: 'https://placehold.co/128x128/63b3ed/ffffff?text=Vue',
                imageAlt: 'Vue.js Placeholder Image',
                linkUrl: 'https://whizmath.com'
            };
        }
    }).mount('#app-vbind');
</script>

`v-on`: Handling Events

Attach event listeners to DOM elements. Shorthand: `@event`.

v-on Example 
<!-- In HTML -->
<div id="app-von">
    <button @click="increment" class="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded-lg">
        Increment
    </button>
    <p class="mt-2">Count: {{ count }}</p>

<script>
    createApp({

üì¶ Components & Props: Building Blocks

Components are reusable Vue instances with a pre-defined set of options. They allow you to encapsulate reusable functionality and UI.

Defining a Component

In Vue, you can define a component using the `component` method or by directly defining it as an object within your `createApp` call. For single-file components (SFCs), you'd use `.vue` files and a build setup, but for CDN, we define them directly.

Defining a Global Component 
<!-- In HTML, use the component name as a custom tag -->
<div id="app-components">
    <my-button></my-button>
    <my-button></my-button>
</div>

<script>
    const app = createApp({}); // Create app instance

    // Define a component
    app.component('my-button', {
        data() {
            return {
                count: 0
            };
        },
        template: `
            <button @click="count++" 
                    class="bg-green-500 hover:bg-green-700 text-white font-bold py-2 px-4 rounded-lg m-2">
                Count: {{ count }}
            </button>
        `
    });

    app.mount('#app-components');
</script>

Props: Passing Data to Components

Props are custom attributes you can register on a component. When a value is passed to a prop, it becomes a property on that component instance.

Component with Props 
<!-- In HTML -->
<div id="app-props">
    <greeting-message name="Whizmath Learner"></greeting-message>
    <greeting-message name="Vue Enthusiast"></greeting-message>
</div>

<script>
    const appProps = createApp({});

    appProps.component('greeting-message', {
        props: ['name'], // Declare the 'name' prop
        template: `
            <p class="text-xl text-yellow-300">
                Hello, <span class="font-bold">{{ name }}</span>! Welcome to Vue.js.
            </p>
        `
    });

    appProps.mount('#app-props');
</script>

‚ö° Computed Properties & Watchers

Computed Properties

Computed properties allow you to define properties that are derived from existing data. They are cached based on their reactive dependencies, meaning they only re-evaluate when their dependencies change. This makes them efficient for complex calculations or data transformations.

Computed Property Example 
<!-- In HTML -->
<div id="app-computed" class="flex flex-col gap-2">
    <input type="number" v-model="price" placeholder="Price" 
           class="shadow appearance-none border rounded w-full py-2 px-3 text-gray-700 leading-tight focus:outline-none focus:shadow-outline">
    <input type="number" v-model="quantity" placeholder="Quantity" 
           class="shadow appearance-none border rounded w-full py-2 px-3 text-gray-700 leading-tight focus:outline-none focus:shadow-outline">
    <p>Total: <span class="font-bold text-green-400">${{ totalPrice.toFixed(2) }}</span></p>
</div>

<script>
    createApp({
        data() {
            return {
                price: 10,
                quantity: 1
            };
        },
        computed: {
            totalPrice() {
                // This will re-evaluate only when price or quantity changes
                return this.price * this.quantity;
            }
        }
    }).mount('#app-computed');
</script>

Adjust the price and quantity above to see the computed `totalPrice` update automatically.

Watchers (`watch`)

While computed properties are for derived values, watchers are useful when you need to perform "side effects" in response to reactive data changes. For example, fetching data when a prop changes, or performing complex DOM manipulations.

Watcher Example 
<!-- In HTML -->
<div id="app-watcher">
    <input type="text" v-model="question" placeholder="Ask a yes/no question..." 
           class="shadow appearance-none border rounded w-full py-2 px-3 text-gray-700 leading-tight focus:outline-none focus:shadow-outline">
    <p class="mt-2">Answer: <span class="font-bold text-orange-400">{{ answer }}</span></p>
</div>

<script>
    const { createApp, watch } = Vue;
    createApp({
        data() {
            return {
                question: '',
                answer: 'I cannot give you an answer until you ask a question!'
            };
        },
        watch: {
            // whenever question changes, this function will run
            question(newQuestion, oldQuestion) {
                if (newQuestion.includes('?')) {
                    this.answer = 'Thinking...';
                    setTimeout(() => {
                        this.answer = Math.random() > 0.5 ? 'Yes!' : 'No!';
                    }, 1000);
                } else {
                    this.answer = 'I cannot give you an answer until you ask a question!';
                }
            }
        }
    }).mount('#app-watcher');
</script>

Type a question ending with '?' into the input above and see the watcher provide an answer.

üö¶ Conditional & List Rendering

Conditional Rendering (`v-if`, `v-else-if`, `v-else`, `v-show`)

Vue provides directives to conditionally render elements based on data.

  • `v-if`: Renders the element and its components conditionally. If the condition is false, the element is completely removed from the DOM.
  • `v-show`: Conditionally displays the element. The element is always rendered in the DOM, but its CSS `display` property is toggled.
Conditional Rendering Example 
<!-- In HTML -->
<div id="app-conditional" class="flex flex-col gap-4">
    <button @click="isLoggedIn = !isLoggedIn" 
            class="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded-lg">
        Toggle Login Status
    </button>
    
    <p v-if="isLoggedIn" class="text-green-400 font-semibold">Welcome, logged in user!</p>
    <p v-else class="text-red-400 font-semibold">Please log in.</p>

    <div v-show="isLoggedIn" class="bg-gray-700 p-3 rounded-md">
        <p class="text-gray-300">This content is shown/hidden with v-show.</p>
    </div>
</div>

<script>
    createApp({
        data() {
            return {
                isLoggedIn: false
            };
        }
    }).mount('#app-conditional');
</script>

Click the "Toggle Login Status" button to see `v-if` and `v-show` in action.

List Rendering (`v-for`)

The `v-for` directive is used to render a list of items based on an array. It requires a unique `key` attribute to help Vue track each node's identity, which is crucial for performance and state consistency.

List Rendering Example 
<!-- In HTML -->
<div id="app-list">
    <h3>Whizmath Learning Path:</h3>
    <ul class="list-disc list-inside bg-gray-700 p-4 rounded-lg">
        <li v-for="lesson in lessons" :key="lesson.id" class="mb-2 text-gray-300">
            {{ lesson.name }}
        </li>
    </ul>
</div>

<script>
    createApp({
        data() {
            return {
                lessons: [
                    { id: 1, name: 'Basic Arithmetic' },
                    { id: 2, name: 'Introduction to Algebra' },
                    { id: 3, name: 'Geometry Fundamentals' },
                    { id: 4, name: 'Calculus Basics' }
                ]
            };
        }
    }).mount('#app-list');
</script>

üí¨ Component Communication

In a component-based architecture, components often need to communicate with each other.

Props Down, Events Up (`emit`)

This is the most common pattern:

  • Parent passes data to child via `props`.
  • Child communicates back to parent via custom `events` using `$emit`.

Parent-Child Communication Example 
<!-- Parent Component (e.g., in your main app instance template) -->
<div id="app-comm" class="bg-gray-700 p-6 rounded-lg">
    <h3>Parent Component</h3>
    <p>Child counter value: <span class="font-bold">{{ childCount }}</span></p>
    <!-- Listen for 'increment' event from child and update childCount -->
    <child-counter @increment="childCount++"></child-counter>
</div>

<script>
    const appComm = createApp({
        data() {
            return {
                childCount: 0
            };
        }
    });

    // Child Component
    appComm.component('child-counter', {
        data() {
            return {
                localCount: 0
            };
        },
        template: `
            <div class="bg-gray-800 p-4 rounded-md mt-4">
                <h4>Child Counter</h4>
                <p>Local count: {{ localCount }}</p>
                <button @click="incrementLocal" 
                        class="bg-teal-500 hover:bg-teal-700 text-white py-1 px-3 rounded-md mt-2">
                    Increment Child
                </button>
            </div>
        `,
        methods: {
            incrementLocal() {
                this.localCount++;
                this.$emit('increment'); // Emit custom 'increment' event
            }
        }
    });

    appComm.mount('#app-comm');
</script>

Click the "Increment Child" button to see both the child's local count and the parent's `childCount` update.

Provide / Inject (for deeper trees)

For scenarios where you need to pass data down through many levels of nested components, `provide` and `inject` offer a more convenient alternative to prop-drilling. A parent component can `provide` data, and any descendant component can `inject` that data, regardless of how deep it is in the component tree.

Provide/Inject Example (Theme) 
<!-- Main App instance (Parent providing theme) -->
<div id="app-theme-context" class="bg-gray-700 p-6 rounded-lg">
    <h3>App (Provider)</h3>
    <button @click="toggleTheme" 
            class="bg-indigo-500 hover:bg-indigo-700 text-white font-bold py-2 px-4 rounded-lg">
        Toggle Theme
    </button>
    <deep-nested-component></deep-nested-component>
</div>

<script>
    const appTheme = createApp({
        data() {
            return {
                currentTheme: 'dark'
            };
        },
        // Provide the theme and a method to toggle it
        provide() {
            return {
                theme: Vue.computed(() => this.currentTheme), // Make theme reactive
                toggleTheme: this.toggleTheme
            };
        },
        methods: {
            toggleTheme() {
                this.currentTheme = this.currentTheme === 'dark' ? 'light' : 'dark';
            }
        }
    });

    appTheme.component('deep-nested-component', {
        template: `
            <div class="bg-gray-800 p-4 rounded-md mt-4">
                <h4>Deep Nested Component</h4>
                <another-nested-component></another-nested-component>
            </div>
        `
    });

    appTheme.component('another-nested-component', {
        // Inject the provided theme and toggle function
        inject: ['theme', 'toggleTheme'],
        template: `
            <div class="bg-gray-900 p-3 rounded-md mt-2">
                <p>Current theme in deeply nested component: 
                    <span :class="{'text-gray-200': theme.value === 'dark', 'text-yellow-400': theme.value === 'light'}" class="font-semibold capitalize">
                        {{ theme.value }}
                    </span>
                </p>
                <button @click="toggleTheme" 
                        class="bg-purple-600 hover:bg-purple-800 text-white py-1 px-3 rounded-md mt-2">
                    Toggle Theme from Nested
                </button>
            </div>
        `
    });

    appTheme.mount('#app-theme-context');
</script>

Click the "Toggle Theme" buttons (both parent and nested) to see how theme data is shared reactively across component levels.

‚öôÔ∏è Advanced Reactivity

Vue 3 introduced the Composition API, which provides a more flexible way to organize and reuse reactive logic, especially in larger components. This contrasts with the Options API (which uses `data`, `methods`, `computed`, `watch` objects) that we've seen so far.

Composition API (`setup`, `ref`, `reactive`)

The Composition API allows you to group related logic together, making it easier to read and maintain large components. It uses `setup()` function and reactivity primitives like `ref()` and `reactive()`.

Composition API Counter Example 
<!-- In HTML -->
<div id="app-composition-api" class="bg-gray-700 p-6 rounded-lg">
    <composition-counter></composition-counter>
</div>

<script>
    const { createApp, ref } = Vue;

    const appComposition = createApp({});

    appComposition.component('composition-counter', {
        setup() { // The entry point for Composition API
            const count = ref(0); // Create a reactive reference

            const increment = () => {
                count.value++; // Access ref value with .value
            };

            // Expose properties and methods to the template
            return {
                count,
                increment
            };
        },
        template: `
            <div>
                <h4>Composition API Counter</h4>
                <p>Count: <span class="font-bold text-yellow-300">{{ count }}</span></p>
                <button @click="increment" 
                        class="bg-blue-500 hover:bg-blue-700 text-white py-2 px-4 rounded-lg mt-2">
                    Increment (Composition)
                </button>
            </div>
        `
    });

    appComposition.mount('#app-composition-api');
</script>

Click the "Increment (Composition)" button to see the counter built with the Composition API.

Custom Directives

While `v-` directives handle most common use cases, Vue also allows you to register your own custom directives to encapsulate DOM manipulation logic.

Custom Directive Example 
<!-- In HTML -->
<div id="app-custom-directive" class="bg-gray-700 p-6 rounded-lg">
    <p v-highlight="highlightColor">This text will be highlighted!</p>
    <input type="color" v-model="highlightColor" class="mt-2">
</div>

<script>
    const appDirective = createApp({
        data() {
            return {
                highlightColor: '#48bb78' // Green by default
            };
        }
    });

    // Register a custom directive named 'highlight'
    appDirective.directive('highlight', {
        // Called when the bound element's attributes are updated.
        // Also called before the element's parent component and its children's VNodes are updated.
        updated(el, binding) {
            el.style.backgroundColor = binding.value;
            el.style.padding = '0.5rem';
            el.style.borderRadius = '0.5rem';
        }
    });

    appDirective.mount('#app-custom-directive');
</script>

Change the color in the input field above to see the custom `v-highlight` directive in action.
